2. Internal Monologue

Text without tools = context, not response. When the LLM writes text without calling a tool, it's treated as internal thinking — appended as an assistant message in the conversation. The LLM sees its own analysis on the next round and builds on it. The user never sees intermediate text. Turn only ends when tools are exhausted, hard ceiling is hit (round 12), or tool limit reached (15).

3. Tool Loop Guardrails (applied every round)

apply_guardrails() — tool_loop_guard.py
Round 2: Parallel nudge — "batch independent tool calls"
Round 3: Think-in-code nudge — "write ONE run_script, not 10 reads"
Round 4: Delegation offer — "use delegate_task for background work"
Round 8: Wrap-up nudge — "finish within 2 more rounds"
Round 12: Hard ceiling — force stop, return partial results
Tool cap: MAX_TOOLS_PER_TURN = 15 — end turn
Token burn: >300K tokens — circuit breaker, respond now
Timeout: 120s aggregate — force stop
Loop detection (_build_guardrail): If same tool returns same error fingerprint 3x in a row → inject LOOP_DETECTED message, forbid retrying that tool. Plan anchor injected on every round: goal + tools used + last error.

6. Crew System (Multi-Agent)

User request with task keywords
"build", "research", "refactor", "analyze"...
▼
Classify

Decompose
LLM creates JSON plan: steps + dependency graph
▼
Execute (parallel by dependency level)
Workers append to shared markdown scratchpad
▼
Worker 1
Researcher (web_search)
Worker 2
Coder (write_file)
Worker 3
Reviewer (read_file)
▼
Merge (free Zen key)
Synthesis of all done steps → final report
Failure isolation: If step fails, dependents skipped. Completed results preserved. Partial merge still delivered. Workers have own SQLite sessions (self-memory via FTS5). Provider routing: per-profile primary + shared pool fallback.
